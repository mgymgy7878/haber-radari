# v0.6.4 — Gerçek Provider Pilot Runbook

**Durum:** Operasyonel hazırlık dokümanı — gerçek LLM çağrısı bu dokümanın parçası değildir.

## Amaç

External LLM digest pilotunu güvenli, gözlemlenebilir ve geri alınabilir şekilde çalıştırmak için operatör checklist, budget görünürlüğü ve rollback prosedürü.

## Pilot öncesi şartlar

- [ ] `main` üzerinde v0.6.3+ altyapı merge edilmiş
- [ ] `GET /api/v1/smart-digest/status` read-only endpoint erişilebilir
- [ ] `pnpm smart-digest:status` evidence JSON üretebiliyor
- [ ] Android cihaz/emülatör API'ye `adb reverse` ile ulaşabiliyor
- [ ] Operatör onay cümlesi hazır (aşağıda)
- [ ] Günlük budget limiti onaylandı (varsayılan: 5)
- [ ] Request başına limit onaylandı (varsayılan: 1)

## Env checklist (local `.env` — asla commit etme)

```env
LLM_DIGEST_ENABLED=1
LLM_DIGEST_PROVIDER=external
LLM_DIGEST_EXTERNAL_ENABLED=0          # pilot anında operatör 1'e çeker
LLM_DIGEST_REQUIRE_OPERATOR_APPROVAL=1
LLM_DIGEST_REQUIRE_CACHE=1
LLM_DIGEST_DAILY_LIMIT=5
LLM_DIGEST_PER_REQUEST_LIMIT=1
LLM_DIGEST_TIMEOUT_MS=8000
LLM_DIGEST_MODEL=<provider-model>
LLM_DIGEST_API_KEY=<local-only-key>
LLM_DIGEST_LOG_PROMPTS=0
LLM_DIGEST_LOG_RESPONSES=0
OPERATOR_APPROVED_REAL_LLM_DIGEST=0    # pilot anında 1 yapılır
```

**Kural:** `.env` dosyası yalnızca local makinede kalır; repo'ya commit edilmez. `.env.example` boş placeholder içerir.

## API key yerleştirme

1. `apps/api/.env` oluştur (gitignore'da olmalı).
2. `LLM_DIGEST_API_KEY=` alanına provider key'i yapıştır.
3. `git status` ile `.env`'in staged olmadığını doğrula.
4. `GET /api/v1/smart-digest/status` → `safety.apiKeyConfigured: true`, key değeri **dönmez**.

## adb reverse gereksinimi

Android emülatör/cihaz localhost API'ye erişmek için:

```powershell
adb reverse tcp:3001 tcp:3001
```

API `http://127.0.0.1:3001` üzerinde çalışırken Android `localhost:3001` kullanabilir.

## Budget limitleri

| Limit | Varsayılan | Env |
|-------|------------|-----|
| Günlük external call | 5 | `LLM_DIGEST_DAILY_LIMIT` |
| Request başına | 1 | `LLM_DIGEST_PER_REQUEST_LIMIT` |

Alanlar: `externalCalls`, `cacheHits`, `failures`, `budgetDenied`, `approvalDenied`, `lastUpdatedAt`

## Operator approval cümlesi

Gerçek external çağrı öncesi açık onay:

```text
v0.6.3 gerçek provider pilotu için 1 çağrıya onay veriyorum
```

Env karşılığı: `OPERATOR_APPROVED_REAL_LLM_DIGEST=1`

Onay yoksa: `OPERATOR_APPROVAL_REQUIRED`, HTTP 200 korunur.

## Tek çağrı pilot akışı

1. **Pre-flight:** `pnpm smart-digest:status` → `realExternalCallAllowedNow: false` (henüz kapalı)
2. Local `.env` key + model doldur (commit etme)
3. Operatör onay cümlesi + `OPERATOR_APPROVED_REAL_LLM_DIGEST=1`
4. `LLM_DIGEST_EXTERNAL_ENABLED=1` (yalnızca pilot anında)
5. API restart
6. Status kontrol: `realExternalCallAllowedNow: true`, `reasonIfBlocked: null`
7. **Tek** smart-feed isteği: `bypassCache=1` ile cache miss zorla (yalnızca 1 eligible item)
8. Digest `status: GENERATED` veya `CACHED` doğrula
9. Budget dosyasında `externalCalls` artışını doğrula
10. Aynı input ile ikinci istek → `CACHED`, provider tekrar çağrılmamalı
11. Pilot bitince: `LLM_DIGEST_EXTERNAL_ENABLED=0`, `OPERATOR_APPROVED_REAL_LLM_DIGEST=0`

## Cache hit doğrulama

- İlk çağrı: `GENERATED`, `modelProvider: external`
- Aynı cluster/metadata ile ikinci çağrı: `CACHED`
- Budget: ikinci çağrıda `externalCalls` artmamalı, `cacheHits` artmalı

## Rollback prosedürü

Acil durumda hemen:

```env
LLM_DIGEST_EXTERNAL_ENABLED=0
OPERATOR_APPROVED_REAL_LLM_DIGEST=0
LLM_DIGEST_ENABLED=0
```

1. API sürecini durdur
2. Env rollback uygula
3. API restart
4. `GET /api/v1/smart-digest/status` → `realExternalCallAllowedNow: false`
5. İsteğe bağlı: `.cache/smart-digest/` pilot cache temizliği
6. Budget dosyasını arşivle (silme — audit için sakla)

## Hata durumları

| errorCode | Anlam | HTTP | Aksiyon |
|-----------|-------|------|---------|
| `OPERATOR_APPROVAL_REQUIRED` | Onay env yok | 200 | Onay ekle veya external kapat |
| `PROVIDER_CONFIG_MISSING` | API key/model eksik | 200 | Local `.env` düzelt |
| `BUDGET_EXCEEDED` | Günlük limit doldu | 200 | Yarın bekle veya limit artır (operatör kararı) |
| `PER_REQUEST_LIMIT` | Request limit aşıldı | 200 | Tek item pilot — beklenen |
| `NOT_ELIGIBLE_FOR_EXTERNAL` | PUBLISH_MAIN dışı / düşük önem | 200 | Normal — external yok |
| `DIGEST_PROVIDER_FAILED` | Timeout/network/provider | 200 | Rollback değerlendir |
| `EXTERNAL_DISABLED` | External kapalı | 200 | Env kontrol |

## Ne yapılmayacak?

- Tam haber metni (`fullText`, `body`, `scrapedText`) LLM'e gönderilmez
- Scraping yok
- Android'de API key yok, mobile LLM çağrısı yok
- Prompt/response log default kapalı (`LLM_DIGEST_LOG_PROMPTS=0`)
- Unlimited batch run yok — günlük 5 / request 1
- Production key repo'ya commit edilmez
- Evidence içine key/env dump girilmez
